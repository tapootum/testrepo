สรุป Spring Training Day 3

February 1st, 2011 Leave a comment Go to comments

Spring Training Day 3

(Slide ครับ SpringTransaction , สปริงเฟรมเวิร์ค3.0 )
ผ่านพ้นไปสำหรับ Spring Training Day 3 เมื่อวันอาทิตย์ที่ 22/11/2009 โดยที่รอบนี้ก็เหมือนรอบที่แล้วคือลุ้นเรื่องสถานที่จนสัปดาห์สุดท้าย แล้วตอนแรกคิดว่าต้องเลื่อนออกไปแต่สุดท้ายก้ได้ @b_architect เข้ามาช่วยชีวืตไว้
โดยการขอสถานที่ที่ KT-ZMICO ให้ต้องขอขอบคุณอีกครับ เพราะนอกจากสถานที่แล้วเรายังได้ขนมและเครื่องดื่มอีกต่างหากทำให้งานดูดีขึ้นมาทันใด
สำหรับงานในครั้งนี้ ผมโดยส่วนตัวรู้สึกว่าราบรื่นกว่าสองครั้งก่อนมากพอสมควร เนื่องจากเราได้แก้ปัญหาหลักๆไปสองข้อคือ

1. Development Environment ถึงแม้ครั้งที่แล้วเราจะใช้ MAVEN เข้ามช่วยแต่ก็ยังช่วยได้ไม่ดีเพราะยังมีปัญหาที่คาดไม่ถึงอีกหลายเรื่องเช่น Database Server ที่ทุกคนใช้นั้น Set มาไม่เหมือนกันทำให้มีปัญหาตอน Test ว่าบางคน Data ไม่เข้า
บางคนสร้างตารางไม่ได้ งานนี้เลยบังคับใช้ Virtual Box ที่ทางทีมงานเตรียมไว้ให้เลย ตามคำแนะนำของ @pphetra และมันก็แก้ปัญหาไปได้เยอะมากทีเดียวเพราะทุกคนใช้ของเหมือนกันหมด ทำให้ทุกคน focus ที่การเขียนโปรแกรมได้เต็มที่
และ TA ก็ไม่ต้องมานั่งแก้ปัญหาบ้าบอนอกเรื่องเหมือนคราวที่แล้ว

2. เรื่อง Technology ที่ใช้ สองครั้งที่แล้วเราพยายามโชว์เทพด้วยการใช้งาน Hibernate และ JPA คือมันเทพจริงๆแต่สำหรับคนที่ไม่เคยใช้มันมาก่อน ผลที่ได้ออกมาคือโลกแตกครับ ทีมงานต้องพบกับ error พิศดารที่เราไม่เคยเจอกันมาก่อน และแน่นอน
หลอนนนนนนน คราวนี้เราเลยเปลี่ยนแนวไปใช้ Spring JDBC และผลที่ได้คิองานดูตรงไปตรงมา ไม่ต้องอธิบายอะไรมาก คนสอนมีความมั่นใจว่าจะไม่หน้าแตกบ่อยจนเกินไป

แต่ปัญหาที่เรายังแก้ไม่ได้คือเนื้อหาตอนเช้าอาจจะมากเกินไปทำให้ไปเบียดบังเรื่อง AOP และ Transaction ให้ตกไปช่วงบ่ายและนั่นหมายความว่าเรื่อง SpringMVC เราก็ยังคงเริ่มตอนบ่ายสี่เหมือนเดิม ==” ทำให้ต้องทำ Lab แห้งกันไปเสียเยอะ

ขอบบคุณ

  1. TA ทุกท่าน @boyone, @somkiat, @yashimaexteen, @9tae, @nuboat ไม่มี TA งานนี้ไม่เกิดครับ
  2. @siros_s ที่มาช่วยสอนเรื่องที่ยากและสำคัญที่สุดใน spring คือ AOP, Transaction
  3. @b_architect ที่หาสถานที่ให้ครับพร้อมอำนวยความสะดวกทุกอย่าง
  4. นักเรียนทุกท่าน

แผนสำหรับ spring training day 4
ตอนนี้ยังต้องสุ่มเสียงจากนักเรียนก่อนว่า
1. ยังอยากให้สอน Basic แบบนี้อีกหรือไม่
2. ถ้าไม่ก็สอน Module อื่นๆของ Spring ไปเลยเช่น Spring Security, Spring Integration, Spring Batch, Spring Web Flow
